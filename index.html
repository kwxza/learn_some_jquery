Welcome!

If you're a new dev wondering whether or not to learn jQuery, you've probably noticed at least one of two things:

  • A lot of advice out there says you don't need jQuery anymore.
  • A lot of JavaScript code out there is chock-full of jQuery.

Should you learn jQuery?

The verdict is still out, but it's usually better to know something and not need it than vice versa. This guide won't make you a jQuery master, but you'll learn just enough to decide whether you want to dig deeper.

Let's learn some jQuery!

Before Starting

This guide assumes that the reader has a working knowledge of:

  • JavaScript
  • CSS
  • HTML
  • The Document Object Model (DOM)

Familiarity with CSS selectors and JavaScript syntax is important, as well as a general understanding of what the DOM is and how it works.

Mozilla Developer Network (MDN) is a great resource to use for exploring more about all of these topics.

What is jQuery?

First released in 2006, jQuery is a JavaScript library that made it easy to create interactive websites that worked consistently in different web browsers. Free to use under the MIT License, jQuery quickly became popular for its simplification of manipulating the DOM, handling events, animating HTML elements and making asynchronous HTTP requests.

Over a decade later, much of what jQuery offers can be done with vanilla JavaScript using the modern DOM API. Despite that, jQuery remains the most widely used JavaScript library, deployed by more than 70% of the top 10 million most popular websites. W3Techs' statistics show that over 20% of these sites use no JS library at all, meaning that almost every website that does use a JS library uses jQuery.

At the time of this writing, the most current stable release of the software is jQuery 3.4.1.

Why jQuery?

Why did jQuery become so popular for so long? To put it simply, using jQuery let developers write less code in less time that worked in most browsers.

Accounting for the differences between the DOM implementations of various browsers is still a concern for developers today, but this was a much more significant issue at the time of jQuery's creation. The depth of the problem is well-illustrated by an example from jQuery's early source code.

The JavaScript code below is checking to see if the HTML document has been fully loaded and parsed by the browser. Note how the code written for each browser is working with the same document object, yet the DOM methods used for each browser are entirely different. 

          
  // If Mozilla is used
  if ( jQuery.browser == "mozilla" || jQuery.browser == "opera" ) {
          // Use the handy event callback
          jQuery.event.add( document, "DOMContentLoaded", jQuery.ready );
  
  // If IE is used, use the excellent hack by Matthias Miller
  // http://www.outofhanwell.com/blog/index.php?title=the_window_onload_problem_revisited  
  } else if ( jQuery.browser == "msie" ) {
  
          // Only works if you document.write() it
          document.write("<scr" + "ipt id=__ie_init defer=true " + 
                  "src=javascript:void(0)><\/script>");
  
          // Use the defer script hack
          var script = document.getElementById("__ie_init");
          script.onreadystatechange = function() {
                  if ( this.readyState == "complete" )
                          jQuery.ready();
          };
  
          // Clear from memory
          script = null;
  
  // If Safari  is used
  } else if ( jQuery.browser == "safari" ) {
          // Continually check to see if the document.readyState is valid
          jQuery.safariTimer = setInterval(function(){
                  // loaded and complete are both valid states
                  if ( document.readyState == "loaded" || 
                          document.readyState == "complete" ) {
  
                          // If either one are found, remove the timer
                          clearInterval( jQuery.safariTimer );
                          jQuery.safariTimer = null;
  
                          // and execute any waiting functions
                          jQuery.ready();
                  }
          }, 10);
  }

Instead of recreating the above functionality to handle the loading of the DOM, a developer using the jQuery library in their project could pass the document as an argument to jQuery's main function jQuery() and then immediately call jQuery's .ready() method on the jQuery object returned from the function.

The library would do the work of filling in the correct browser methods and the developer would have written only one line of code: jQuery(document).ready()

Though modern browsers implement the DOM much more consistently, using jQuery still allows developers to write less code in less time than they otherwise might while maintaining cross-browser compatibility. In fact, jQuery 1.12 is still in use today to support old versions of Internet Explorer.

Setting Up jQuery

jQuery offers both a compressed (minified) and uncompressed version of the software. The uncompressed version is typically used during development, while the compressed version of the library is used once the application is put into production.

You must link a copy of jQuery's software to your project before you can take advantage of its methods. You can use the most current version or any of the previous releases that are still supported. There are several ways to add the jQuery library to your projects.

  • Linking to a CDN

    Using a CDN (Content Delivery Network) is the easiest way to include jQuery in your projects. Simply add a <script> tag to your HTML file with the src attribute set to the CDN link.

    Official CDN links for jQuery can be found at code.jquery.com. The jQuery CDN offers links to compressed and uncompressed versions of both current and past releases, as well as supporting Subresource Integrity (SRI) for website security.

    The <script> tag to include the uncompressed CDN version of jQuery 3.4.1 looks like this: 

                  
  <script
    src="https://code.jquery.com/jquery-3.4.1.js"
    integrity="sha256-WpOohJOqMqqyKL9FccASB9O0KwACQJpFTUBLTYOVvVU="
    crossorigin="anonymous"
  ></script>

  • Downloading jQuery Manually

    Setting up jQuery by downloading it to your local environment is also pretty simple. Download your desired version from jquery.com/download to your project folder.

    Linking to a local version of jQuery with a <script> tag will have the following format where {path} is the path leading to your project folder and {version} is the suffix of the jQuery file being linked to. 

                  
  <script
    src="{path}/jquery-{version}.js"
  ></script>

  • Downloading with a Package Manager

    If you use a package manager, jQuery can be installed from the command line.

    To install jQuery using npm, navigate to your project folder in your command line interface and run the command npm install jquery

    This will install both the compressed and uncompressed versions of jQuery 3.4.1 in your project's node_modules folder.

    Now you can use the uncompressed version by setting node_modules/jquery/dist/jquery.js as the src attribute for your <script> tag. Use jquery.min.js for the compressed version.

    Alternatively, you can import jQuery directly into your project's main JS file by adding the statement import $ from "jquery";

    Using import to add jQuery to your project as a module can be tricky. Most likely, you will require the use of a JavaScript transpiler and a module bundler. Importing modules is an ES6 feature that is not yet well-supported in all browsers. You can check browser compatibility at MDN's documentation page of the import statement.

    A JavaScript transpiler will convert ES6+ syntax like import into backwards-compatible ES5 JavaScript code that will work in most browsers. Babel is one of the most popular ones. Follow these instructions to set up Babel using npm.

    A package downloaded from npm is simply a directory of JS modules, which get installed to your project's node_modules folder. Using modules in node works easily, but browsers cannot find module dependencies on their own. A module bundler will bundle the modules your project depends on into a script file that browsers can read. One of the most commonly-used module bundlers is webpack. Follow these instructions to set up webpack using npm.

Now that you know how to set up jQuery, here is an example of some jQuery code. 

          
  $(document).ready(function(){
    $("button").click(function(){
        $("div").animate({height:"toggle"}).append("hi");
    });
  });

Note the frequent use of the '$' sign, followed by references to HTML elements within parentheses. This brings us to the first important jQuery concept - the jQuery function.

The jQuery Function

The jQuery function allows developers to precisely select any element or collection of elements within an HTML document. jQuery() can also create elements if passed a string of HTML as its argument. The jQuery function always returns a jQuery object. jQuery objects will be covered in more detail in Learn Some jQuery: jQuery Objects.

Working with jQuery():

  • Handling the loading of the DOM:

    To safely manipulate the DOM, it is necessary to wait until the HTML document has been fully loaded and parsed by the browser. With jQuery, you can handle DOM loading by wrapping all code that depends on the readiness of the DOM within a call to jQuery().

    This can be done by putting your DOM-dependent code into a callback function that can be passed as an argument to jQuery(). This is normally done with an anonymous function, but you can also define a named function containing your code manipulating the DOM. Then, simply pass the function name to jQuery().

    You will commonly see $() used in place of jQuery(). We will cover the '$' syntax in more detail in Learn Some jQuery: The '$' Sign. Let's use that syntax now to look at an example of handling the loading of the DOM. 

                  
  /*              
   Handling DOM loading with jQuery()
   and an anonymous callback function.
  */
  $(function() {
    // Code manipulating the DOM...
  });
  

  /*
   Does the same as above, but with
   a named callback function.
  */
  // Defining the callback function
  function runOnceDOMIsReady() {
    // Code manipulating the DOM...
  }
  // Wrapping the callback function in jQuery().
  $(runOnceDOMIsReady);

    The example above shows the recommended way to handle the loading of the DOM. When looking at code using jQuery, you will often see another way of handling DOM loading, which makes use of jQuery's .ready() method: 

                  
  $(document).ready(function() {
    // Code manipulating the DOM...
  });

    This form of handling DOM loading has been deprecated to avoid misunderstanding of how the .ready() method works. The syntax of the statement $(selector).ready() implies that the execution of the .ready() method depends on the loading of the element(s) selected by jQuery(selector). The selection is actually irrelevant. The .ready() method will always execute as soon as possible once the entire DOM has finished loading.

  • Wrapping DOM elements in jQuery objects:

    The jQuery function is typically used to select elements from the DOM and wrap them within a newly-created jQuery object. This allows jQuery's object methods to be used in place of the native DOM methods. Working with DOM elements directly is possible, but can be difficult to do concisely while maintaining browser-compatibility.

    jQuery() takes a selector expression in order to select elements from the DOM. Any and all elements that match the selector will be found and returned by the function. Keep in mind that jQuery() will return an empty jQuery object if there are no DOM elements that match the selector. Many of the possible selector expressions in jQuery are identical to CSS selectors. We will cover selector expressions in more detail in Learn Some jQuery: Selecting Elements.

  • Creating new DOM elements:

    The jQuery function is also used to create new DOM elements. This is done by giving a string of valid HTML to jQuery(). The function returns a jQuery object containing the newly-created elements, which can be inserted into the document using jQuery's object methods. It is possible to create elements belonging to a document other than the one in which the jQuery library has been loaded.

    To create an element, you can provide jQuery() with either:

    • a string describing a single HTML element which can have text content and attributes,
    • a more complex string containing multiple HTML elements that may be nested within each other, or
    • a string describing a simple HTML element and an object describing its attributes.

    In most cases, jQuery will create a new <div> element and set its innerHTML property to the HTML string passed to the function.

    Below are a few examples showing how to use jQuery() to create elements. The examples will use the $() syntax. 

                  
  /*
    Creating a <h1> with text content and 
    an id attribute and adding it to the body.
  */
  $('<h1 id="header">Hello World</h1>').appendTo('body');


  /*
    Creating an <img> element nested 
    within an <a> element and saving 
    the created elements to a named 
    variable as a jQuery object.
  */
  const $imgLink = $('<a><img></a>');


  /*
    Creating an <input> element with its
    type attribute set to 'checkbox'
    and appending it to the element
    with the id of 'form'.
  */
  $('<input type="checkbox">').appendTo('#form');


  /* 
    Creating a <p> element that belongs
    to the parent document of the 
    element with the id of 'myiframe'.
    (The <iframe> element embeds another
     HTML document within the current one,
     representing a nested browsing context.)
  */
  const $iframeParagraph = $(
    '<p>hello iframe</p>', 
    $('#myiframe').prop('contentWindow').document
  );


  /*
    Creating a <div> element with an
    object describing attributes, event
    handlers and jQuery methods to call
    on the created element.
    The created <div> is wrapped in a
    jQuery object and saved to a variable.
    Object keys:
      class - Sets the 'class' attribute.

      text -  Calls jQuery's .text() method 
              once the <div> has been 
              created - this sets the text
              content of the element.
              
      click - Adds an event handler to the
              <div>, which will run the 
              provided callback function when
              the element has been clicked.
  */
  const $testDiv = $('<div/>', {
    class: 'test',
    text: 'Click me!',
    click: function() {
      $(this).toggleClass('test');
    }
  });

Possible Arguments for jQuery():

  • Selection Arguments

    There are various possible arguments for wrapping a selection of pre-existing DOM elements into a jQuery object. A selector expression for jQuery()'s argument finds all the matching elements present in the DOM. The argument can also be a specific element or collection of elements, given either as DOM elements or as another jQuery object. jQuery() is even able to create an empty jQuery object or wrap a JavaScript Object. Here is a list of the possible arguments for selections:

    1. jQuery(selector [,context])
      • selector - a string containing a selector expression.
      • context (optional) - a DOM element, document or jQuery object as the context in which to find any element(s) matching the selector. The default context is the entire document in which the jQuery library has been loaded.
    2. jQuery(element)
      • element - a DOM element.
    3. jQuery(elementArray)
      • elementArray - an array of DOM elements.
    4. jQuery(object)
      • object - a PlainObject, which is jQuery's definition of a JavaScript object of type Object.
    5. jQuery(selection)
      • selection - a jQuery object to be cloned, containing previously selected/created elements.
    6. jQuery()
      • Providing no arguments to the function will return an empty jQuery object.

  • Creation Arguments

    There are only a few possible arguments for creating new DOM elements. Keep in mind that created elements do not appear in the DOM until they have been inserted into the document. The created elements are wrapped within a new jQuery object. Here are the possible creation arguments for jQuery():
    1. jQuery(html [,ownerDocument])
      • html - a string of HTML to be created.
      • ownerDocument (optional) - an HTML document to which the new element(s) will belong. The default document is the HTML document in which the jQuery library has been loaded.
    2. jQuery(html, attributes)
      • html - a string defining a single, standalone HTML element.
      • attributes - a PlainObject containing key-value pairs of attributes, events and methods to call on the newly-created element.

Refer to the jQuery API documentation for more details on how jQuery() works for each of these different arguments.

The '$' Sign

The '$' sign is a recognizable characteristic of jQuery, but has no intrinsic meaning in and of itself. It is simply an alias for 'jQuery' that makes it faster to call the jQuery function.

Thus, jQuery(document).ready() can be written as $(document).ready(). This is the form that you will usually see. In this guide, jQuery() is used primarily to refer to the jQuery function in text, while $() is mostly used within code examples.

Conflicts with other libraries:

The '$' sign is an unused character in JavaScript and there are several JS libraries that use it as an alias for their identifier. Using such a library simultaneously with jQuery can cause a conflict.

You can avoid this by immediately calling jQuery.noConflict() before using the jQuery function to handle the loading of the DOM. This will put jQuery into no-conflict mode. You can then use jQuery() in place of $(), allowing the '$' sign to be used as the other library's alias. The following example shows how this would appear. 

          
  // Putting jQuery into no-conflict mode.
  jQuery.noConflict();

  // Handling DOM loading with jQuery().
  jQuery(function() {
    // jQuery code using jQuery()...
  });

  /*
   Code using '$' as the alias for
   another library can follow here.
  */

You can set a new alias for jQuery() when you enable no-conflict mode. You will then be able to handle DOM loading by calling the jQuery function using the new alias. This is demonstrated below. 

          
  /*          
   Putting jQuery into no-conflict
   mode and setting a new alias.
  */ 
  const $j = jQuery.noConflict();

  /*
   Handling DOM loading using the new alias
   and an anonymous callback function.
  */
  $j(function() {
    // jQuery code using the $j() alias...
  });

  /*
   Code using '$' as the alias for
   another library can follow here.
  */

If you prefer using the $() alias, you can put jQuery into no-conflict mode and handle DOM loading using jQuery(), passing '$' as an argument to the handler's callback function. This allows safe use of the $() alias for all jQuery code within the scope of the callback function. Code outside of the DOM-loading handler will be able to use the '$' alias without any conflict. An example is given below. 

          
  // Enabling no-conflict mode.
  jQuery.noConflict();

  /*
    Handling DOM loading using jQuery()
    and passing '$' to the callback 
    function, to be used within as the 
    alias for jQuery.
  */
  jQuery(function($) {
    // jQuery code safely using $() alias... 
  });

  /*
   Code using '$' as the alias for
   another library can follow here.
  */

The '$' sign in variable names:

In the above example of how to set a new alias for jQuery(), the new alias contains the '$' sign. This is not required, but it is conventional to add the '$' sign as a prefix when naming variables containing jQuery objects. This is to distinguish them from variables containing DOM elements or regular JavaScript data types. 

          
  /*
    Getting and storing the footer 
    element using DOM methods vs jQuery.
  */

  // DOM method.
  const footer = document.getElementsByTagName('footer');

  //jQuery method with naming convention.
  const $footer = $('footer');

In the above example, the variables are both named footer, but the variable containing the jQuery object is prefixed by the '$' sign. This makes it easy to tell which variable refers directly to the DOM element as opposed to the jQuery object wrapping the DOM element, while still making it clear that the element in question is the <footer>.

This pattern is common when working with jQuery. While many jQuery methods return jQuery objects, there are methods which return other types of data, especially DOM elements. Keeping track of which data types are contained in your variables will help avoid introducing errors into your code. Native DOM methods will not work on jQuery objects and jQuery object methods will not work on DOM elements.

Let's go deeper into the topic of jQuery objects.

jQuery Objects

jQuery objects are created when jQuery() is used to select or create DOM elements. They have some similarities to JavaScript Array objects; they have a length property and a zero-based index for the collection of DOM elements they contain. However, jQuery objects cannot be manipulated with native Array methods, such as Array.forEach() or Array.join(). It is best to use jQuery methods when working with jQuery objects.

Working with jQuery objects:

The examples in this section will demonstrate ways of working with jQuery objects. You will see the use of various jQuery object methods as well as different types of selector expressions. If you want to cover selectors in more detail before this section, skip ahead to Learn Some jQuery: Selecting Elements. If you want to know more about a particular object method, search the name of the method in the jQuery API for detailed documentation.

  • Using jQuery object properties:

    jQuery objects possess two properties, both of which can be quite useful.

    • The .jquery property:

      You can check the .jquery property to determine whether a particular object is a jQuery object. A jQuery object will return a string containing the jQuery version number; checking the .jquery property on anything other than a jQuery object will return undefined.

      In JavaScript, strings are true and undefined is false when evaluated in a Boolean context. This means that the .jquery property can be used as a boolean expression for conditional logic. 

                        
  /*
   Choose what to do depending on
   if 'obj' is a jQuery object.
  */
  if(obj.jquery) {
    // Do something... 
  } else {
    // Do something else...
  }

      This can be useful in various scenarios, such as when iterating through a JavaScript Object or Array that contains various types of data which may include jQuery objects.

    • The .length property:

      Selecting DOM elements using jQuery() creates a jQuery object that contains all the elements matching the selector expression given to the function. An easy way to check how many elements are contained in a jQuery object is to use the object's .length property.

      For example, after selecting all the <h1> elements in the document, you may want to see how many there actually are: 

                        
  /*
    Selecting all the <h1> 
    elements in the document.
  */
  const $headings = $("h1");
  
  /* 
    Checking how many <h1>
    elements were selected.
  */
  console.log($headings.length);
  /* 
    Console output:
    An integer that is greater
    than or equal to zero.
  */

      If the document contains no <h1> elements, the .length property will return zero. Calling jQuery() will alway return a jQuery object, however the object will be empty if the provided selector does not match any elements. Checking the .length property is an easy way to verify if the selector expression successfully matched any elements.

  • Characteristics of jQuery objects:

    Each jQuery object is a unique wrapper around the DOM elements that it contains. No two jQuery objects are equal, even if one was cloned from the other. It is a mistake to assume that two jQuery objects are equivalent based on their .length properties or the selector expression used to create them.

    • jQuery objects don't update with the DOM:

      A jQuery object represents the matched elements present in the DOM at the time the object was created. Adding or removing DOM elements does not change the collection within the object, even if the added/removed elements match the object's selector. A jQuery object may contain more, fewer or even different DOM elements than another object created with the same selector.

      The below example demonstrates how two jQuery objects created with the same selector can contain a different number of elements, due to DOM manipulation that occured between their times of creation. 

                        
  /*
   Selecting all <div> 
   elements in the DOM. 
  */
  const $divCollection1 = $('div');
  
  /*
   Removing the last <div>
   on the page using jQuery's 
   .remove() method.
  */
  $('div:last-of-type').remove();

  /*
   Selecting all <div> 
   elements again.
  */
  const $divCollection2 = $('div');

  /*                 
   Comparing the .length property
   of the two jQuery objects.
  */
  console.log($divCollection1.length == $divCollection2.length);
  /* 
    Console output: false 

    The .length property of the second object
    is different from the first, even though
    they were created with the same selector.
  */

      The contents of the first object $divCollection1 remain the same after the last <div> element in the DOM has been removed. Comparing the .length property of $divCollection1 to $divCollection2 shows that they are not the same size - meaning $divCollection1 contains a element that no longer exists in the DOM.

      This is an important point - jQuery objects can hold elements that do not appear in the DOM. Similarly to how changes in the DOM do not update jQuery objects, DOM elements created with jQuery() do not appear in the DOM until they are inserted into the document.

    • jQuery objects don't sort their contents:

      The indexing of DOM elements within a jQuery object matches the order that the elements appeared in the DOM at the time the object was created. Two jQuery objects created with the same selector can contain the same DOM elements arranged in a different order.

      To demonstrate this, the following example will use the HTML structure of this web page. This page consists of 12 <section> elements. Each section has an id attribute corresponding to the title of the section. The example will create two jQuery objects containing all the <section> elements on the page. The second object will be created after the "Welcome!" section of the page has been moved from its original position in the DOM. 

                        
  /*
    Creating a jQuery object of all 
    the <section> elements before 
    manipulating the DOM.
  */
  const $sectionsBefore = $('section');

  /*
    Using jQuery's .insertAfter() method 
    to place the "Welcome!" section  after 
    the "jQuery Objects" section.
  */
  $('#Welcome').insertAfter('#jQuery_Objects');

  /*
    Creating a jQuery object of all the <section> 
    elements after manipulating the DOM.
    Then, comparing its .length property to the
    .length property of $sectionsBefore.
  */
  const $sectionsAfter = $('section');
  console.log(
    'Are the .length properties the same? '
    + ($sectionsBefore.length == $sectionsAfter.length)
  );
  /*
    Console output:
    Are the .length properties the same? true
  */

  /*
    Using jQuery's .index() method to 
    log the index of "#Welcome" in
    each object. 
  */
  console.log($sectionsBefore.index($('#Welcome')));
  console.log($sectionsAfter.index($('#Welcome')));
  /* 
    Console output: 0 
    Console output: 6 
  
    The "Welcome!" section appears at different
    indexes in the two jQuery objects.
  */

      As can be seen from the example above, no elements were added or removed from the DOM. The two jQuery objects have the same .length property and hold references to the same DOM elements, yet changing the position of the "Welcome!" section changed the ordering of the object created after the DOM manipulation.

    Comparing jQuery objects and their properties has limited utility. You will usually need to access a jQuery object's elements directly if you need to make comparisons.

  • Accessing DOM elements within a jQuery object:

    There are several ways to access DOM elements within jQuery objects. The methods differ depending on if you want to access a specific element, a collection of elements or isolate a specific element as a jQuery object.

    • .get():

      When called on a jQuery object, the .get() method takes a positive or negative index as an argument and returns the DOM element at that index. Let's use $size to refer to the number of elements within a jQuery object. Positive indexes start at 0 for the first element of the collection and go up to $size - 1. Negative indexes step backwards from the last element, starting with -1 and going down to -$size. Any index values outside of these bounds will return undefined.

      Let's demonstrate this by again using a selection of the 12 <section> elements on this page: 

                        
  const $sections = $('section');
  /* 
    'size' will be the number of elements
    within $sections - we know there are 12.
  */
  const size = $sections.length;

  /*
    Index bounds: -$size <= index <= $size - 1
    
    Anything outside the bounds is undefined.

    Let's log the id attributes of the  first and last 
    elements in $sections using postive and negative 
    indexes expressed in terms of the value of $size. 
    Let's also log the result of using .get() with 
    postive and negative indexes that are out of bounds.
  */

  /*
    The console.log statements will use the ES6 template
    literal syntax. Don't confuse the ${} operator of 
    template literals with the $() of jQuery.
    Template literals use ${} to enclose an embedded
    JavaScript expression within a String.
  */
  
  /*
    Positive indexes: 0 <= index <= $size - 1
  */
  console.log(`$size: ${$size}`);
  console.log(`index: 0, section: ${$sections.get(0).id}`);
  console.log(`index: $size - 1, section: ${$sections.get($size - 1).id}`);
  /*
    Console output:
    $size: 12
    index: 0, section: Welcome
    index: $size - 1, section: Why_Not_jQuery
  */

  /*
    Negative indexes: -$size <= index <= - 1
  */
  console.log(`$size: ${$size}`);
  console.log(`index: -1, section: ${$sections.get(-1).id}`);
  console.log(`index: -$size, section: ${$sections.get(-$size).id}`);
  /*  
    Console output:
    $size: 12
    index: -1, section: Why_Not_jQuery
    index: -$size, section: Welcome
  */

  /*
    Positive out-of-bounds: index >= $size 
    Negative out-of-bounds: index <= -$size - 1 
  */
  console.log(`$size: ${$size}`);
  console.log(`index: $size, section: ${$sections.get($size)}`);
  console.log(`index: -$size - 1, section: ${$sections.get(-$size - 1)}`);
  /* 
    Console output:
    $size: 12
    index: $size, section: undefined
    index: -$size - 1, section: undefined
  */

      Be mindful that the .get() method returns the DOM element unwrapped from the jQuery object. You must use native DOM methods to work with the returned element. If you prefer using jQuery object methods, you can use jQuery() to wrap the returned element in a jQuery object.

    • .toArray():

      The .toArray() method does not require any arguments; it simply returns an Array containing all the DOM elements within a jQuery object. This array can then be manipulated using JavaScript Array methods. 

                        
  /*
    Selecting all the internal links that appear
    on the page, but not in the nav menu -
    ie. all the <a> elements with an href 
    attribute beginning with '#' that appear
    inside of a <section> element.

    Then, converting the created object's contents to 
    an array and using the JS method Array.forEach() 
    to display the array index and the .innerText DOM 
    property of each link to the console.
  */                  

  const internalLinksOnPage = $('section a[href^="#"]');

  internalLinksOnPage.toArray().forEach(function(link, index) {
    console.log(`Link at array[${index}]: ${link.innerText}`);
  });
  /*
    Console output:    
    Link at array[0]: Learn Some jQuery: jQuery Objects
    Link at array[1]: Learn Some jQuery: The '$' Sign
    Link at array[2]: Learn Some jQuery: Selecting Elements
    Link at array[3]: Learn Some jQuery: Selecting Elements
  */

      Note that after selecting elements using jQuery(selector), the example converts the created jQuery object's contents into a JavaScript Array and then uses JavaScript methods. It's not uncommon to see jQuery and JavaScript methods used together. If you're wondering whether the same selection can be made with pure JavaScript - it can! The DOM method .querySelectorAll() can take any valid CSS selector as its argument. This returns a NodeList of the matched elements, which is very similar to an Array and can be iterated through using the .forEach() method.

    • Array subscripting:

      jQuery objects possess some Array-like behaviour, including array subscripting. Similar to a JavaScript Array, you can use bracket notation to access an element by its index within a jQuery object. As a consequence, array subscripting lacks the negative-indexing capability of the .get() method. Using a negative index will return undefined.

      Consider a generic jQuery object named $object. Instead of getting the last element by using $object[-1], you would have to use the .length property - ie. $object[$object.length - 1]. This is a lot more verbose than using $object.get(-1) to retrieve the last element. 

                        
  const $sections = $('section');

  /*
    Accessing the first and last elements of
    $sections using array subscripting.
  */                  
  console.log(`Index: 0, section: ${$sections[0].id}`);
  console.log(
    `Index: ${$sections.length - 1}, section: ${
      $sections[$sections.length - 1].id
    }`
  );
  /*
    Console output:
    Index: 0, section: Welcome
    Index: 11, section: Why_Not_jQuery
  */

      Array subscripting returns the DOM element at the specified index. Using DOM methods are required for working directly with the returned element. Once again, if you prefer using jQuery object methods, you can wrap the element by using jQuery().

    • .eq():

      Using .get() or array subscripting lets you extract a specific element from a jQuery object, but what if you want to use jQuery object methods instead of native DOM methods? It's possible to re-wrap the element with jQuery(), but a quicker way is to use the .eq() method.

      Similarly to .get(), the .eq() method takes a positive or negative zero-based index as its argument. However, instead of returning the DOM element itself, .eq() returns a new jQuery object containing only the element at the specified index. jQuery object methods can be immediately used without having to call jQuery(). Let's use the .jquery property in an example to confirm this. 

                        
  /*
    Selecting every link on the page that
    links to an outside location - 
    ie. every <a> element that does NOT 
    have an href attribute beginning with '#'.

    Then, reducing the collection to a jQuery
    object containing the first element only
    and logging its .jquery property.
  */
  const $linksToOutside = $('a:not(a[href^="#"])');
  console.log($linksToOutside.eq(0).jquery);
  /*
    Console output: 3.4.1

    Recall that the .jquery property returns the 
    jQuery version number if the object is a jQuery
    object, otherwise it returns 'undefined'.
  */

      Unlike .get(), calling .eq() with an out-of-bounds index does not return undefined. The .eq() method is actually a filtering method and so will return an empty jQuery object with a length property of zero. In jQuery, filtering methods return an empty object if no element matches their filter condition. In this case, the method filters for any element at the specified index. For more information about jQuery's filtering methods, visit jQuery's API documentation for filtering.

  • Iterating over jQuery objects:

    We just looked at jQuery's .toArray() method, which returns an array of DOM elements that can be manipulated using JavaScript Array methods. If you're using .toArray() so that you can iterate through the elements, you can actually use jQuery's .each() method. The method takes a jQuery object and executes a callback function on each element in the collection. The DOM element is unwrapped from the object within the body of the callback function, JavaScript methods can be used without conversion. The keyword this will allow you to access the current element of the iteration. Take a look at the example below. 

                  
  /*
    Selecting each <h1> element on
    the page and iterating through them.
  */              
  $('h1').each(function() {
    // Uppercasing each <h1> element's text.
    this.innerText = this.innerText.toUpperCase();
  });

    Note how inside the callback function, the native DOM property HTMLElement.innerText and the JavaScript method String.toUpperCase() are used instead of jQuery methods. This is because the .each() method returns each element of the collection as itself - ie. as a DOM element unwrapped from the jQuery object. Thus, native methods are required. You may want to wrap the current element of iteration within a jQuery object so that you can use jQuery methods. This can be done by calling jQuery() with the this keyword as its argument.

    You might also want to access the index of the element currently being iterated. The .each() method automatically passes the index of the current element to the callback function. You can access this index if you name a parameter in the callback function's arguments and use that parameter name to refer to the index.

    The following example will demonstrate using the index of the current element and wrapping the current element in a jQuery object. 

                  
  /*
    Selecting each <a> element on
    the page and iterating through them,
    as well as naming a parameter to use
    for referring to the current index.
  */              
  $('a').each(function(index) {
    /*
     Wrapping each <a> element in a jQuery
     object and getting its href attribute 
     using jQuery's .attr() method.
     Then, using the index to log the link's
     number on the page and its href attribute 
     to the console using ES6 template string syntax.
    */
    const $url = $(this).attr('href');
    console.log(`Link ${index + 1}: ${$url}`);
  });
  
  /*
    Console output:
    Link 1: href of first <a> element on page...
    Link 2: href of second element...
    Link 3: ...
    .
    .
    .
  */
