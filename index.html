Welcome!

If you're a new dev wondering whether or not to learn jQuery, you've probably noticed at least one of two things:

  1. A lot of advice out there says you don't need jQuery anymore.
  2. A lot of JavaScript code out there is chock-full of jQuery.

Should you learn jQuery?

The verdict is still out, but it's usually better to know something and not need it than vice versa. This guide won't make you a jQuery master, but you'll learn just enough to decide whether you want to dig deeper.

Let's learn some jQuery!

Before Starting

This guide assumes that the reader has a working knowledge of:

  • JavaScript
  • CSS
  • HTML
  • The Document Object Model (DOM)

Familiarity with CSS selectors and JavaScript syntax is important, as well as a general understanding of what the DOM is and how it works.

Mozilla Developer Network (MDN) is a great resource to use for exploring more about all of these topics.

What is jQuery?

First released in 2006, jQuery is a JavaScript library that made navigating and manipulating the DOM of an HTML document easy compared to pre-ES6 JavaScript. Free to use under the MIT License, jQuery quickly became popular for its cross-browser compatibility and its simplification of handling DOM events, animating HTML elements and making HTTP requests with Asynchronous JSON and XML (AJAX).

Over a decade later, much of what jQuery offers can be done with vanilla JavaScript using the modern DOM API. Despite that, jQuery remains the most widely used JavaScript library, deployed by over 70% of the top 10 million most popular websites. W3Techs statistics show that over 20% of websites use no JS library at all, meaning that almost every website that uses a JS library uses jQuery.

At the time of this writing, the most current stable release of the software is jQuery 3.4.1.

There are several ways to add jQuery to your projects:

  • Using a CDN

    Using a CDN (Content Delivery Network) is probably the easiest way to include jQuery in your projects. Simply include a <script> tag in your HTML file with the src attribute set to the CDN link.

    Official CDN links for jQuery can be found at code.jquery.com. The jQuery CDN offers links to compressed and uncompressed versions of both current and past releases, as well as supporting Subresource Integrity (SRI) for website security.

    The <script> tag to include the uncompressed CDN version of jQuery 3.4.1 looks like this: 

      <script
    src="https://code.jquery.com/jquery-3.4.1.js"
    integrity="sha256-WpOohJOqMqqyKL9FccASB9O0KwACQJpFTUBLTYOVvVU="
    crossorigin="anonymous"></script>

  • Downloading jQuery Locally

    Setting up jQuery by downloading it to your local environment is also pretty simple. Download your desired version from jquery.com/download to your project folder.

    Add a <script> tag to your HTML with the src attribute set to the path leading to your project folder.

    Linking to a local version of jQuery with a <script> tag will have the following format where {path} is the path to your project folder and {version} is the suffix of the jQuery file being linked to. 

      <script
    src="{path}/jquery-{version}.js"
  >
  </script>

  • Using a Package Manager

    If you use a package manager, jQuery can be installed from the command line.

    To install jQuery using npm, navigate to your project folder in the command line and run the command npm install jquery

    This will install both the compressed and uncompressed versions of jQuery 3.4.1 in your project's node_modules folder.

    You can then use {path}/node_modules/jquery/dist/jquery-{version}.js as the src attribute for your <script> tag.

    You can also import jQuery directly into your project's main JS file by adding the statement import $ from "jquery"

    Be aware that importing modules is an ES6 feature that may not be supported in all browsers. You can check browser compatibility at MDN's documentation page of the import statement.

    If your project will be running in older browsers, using import will require Babel, a JavaScript compiler that converts ES6+ syntax into backwards-compatible JavaScript code. Follow these instructions to set up Babel using npm.

Now that you know how to set up jQuery, here is an example of some jQuery code. 

  $(document).ready(function(){
    $("button").click(function(){
        $("div").animate({height:"toggle"}).append("hi");
    });
  });

Note the frequent use of the $ symbol, followed by HTML element names within parentheses.
This brings us to the first important jQuery concept: the jQuery function.
